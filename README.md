# Catalogue Music and Streaming

This repository contains the modelling code, reported model outputs, and predictive-check materials for the paper *Is Older Music Becoming Dominant in the Age of Streaming?*

The analysis uses a Bayesian generalised linear mixed model estimated with a custom Python implementation of integrated nested Laplace approximation (INLA). The response is `log(1 + Streams)`. The model includes standardised month and track age, country indicators, interactions between track age, month and country, and a track-level random intercept.

The underlying Luminate data are proprietary and are therefore not included. The cached fitted model is provided so that the estimated posterior state can be inspected without distributing the source data.

## Breakdown of each file

### `Model Diagnostics and Robustness.docx`

Provides the supplementary discussion of the prior predictive check, posterior predictive check and robustness assessment. This repository includes the document and the files associated with its prior and posterior predictive checks.

### `Prior_predictive.py`

Runs the prior predictive check using the same covariate structure, fixed-effect priors, random-effect prior and mixed-Gaussian observation model as the main analysis.

The script loads the proprietary modelling data, reconstructs the model design, fits the pilot likelihood used by the final specification, draws parameters from the priors, and simulates replicated outcomes. It saves the prior predictive plot and the numerical summary used to assess whether the prior-likelihood combination generates plausible stream-count distributions.

### `prior_predictive_summary_cyear.csv`

Contains the numerical summary from the prior predictive check. It records the observed and replicated distribution summaries, prior predictive probabilities, likelihood summaries, prior settings and fitted mixture-likelihood parameters.

### `prior_predictive_check_cyear_posterior_predictive_check_combined.png`

Contains the combined prior and posterior predictive-check figure included in `Model Diagnostics and Robustness.docx`. The left panel compares the observed outcome density with simulations from the prior predictive distribution. The right panel compares it with simulations from the fitted posterior predictive distribution.

### `GLMM_run.py`

Defines and runs the main model reported in the paper. It loads the modelling variables from `../Data/main_data_complete_cyear.csv`, removes observations with missing model fields or negative track ages, assigns integer track identifiers, standardises month and track age, creates the country indicators and interaction terms, and specifies the fixed-effect priors.

Before fitting the final model, it fits a Gaussian pilot model to a random sample of tracks. The demeaned pilot residuals are used to estimate the parameters of the mixed-Gaussian likelihood. It then fits the full model through `INLA_solver.py`.

The fitted model is stored in `model-cache/inla-cache.pkl`. The script writes the main fixed-effect estimates to `fixed_effects.csv` and contains the posterior predictive routines used for the combined predictive-check figure. Other intermediate outputs generated by the script are not included.

### `GLMM_run_one_country.py`

Fits the same model separately for each country. Month and track age are standardised using the full-sample means and standard deviations so that the country-specific estimates remain comparable.

For each country and complete year, the script combines the track-age coefficient with the track-age-by-month interaction and calculates a 95% credible interval using the fitted covariance matrix. The collated estimates are written to `country_slopes_pooled.csv`.

When run from the proprietary source data, country-level fitted states are cached in `model-cache-country/`. Those intermediate caches are not included because the reported country-year estimates are provided directly in `country_slopes_pooled.csv`.

### `GLMM_rand.py`

Defines the random-intercept component of the generalised linear mixed model. It converts track identifiers to consecutive group indices, constructs the grouped score and precision terms used by the solver, and evaluates the Gaussian random-effect density.

It also defines the Gamma prior on the random-effect precision, with shape 1 and rate 0.00005, as specified in the paper.

### `INLA_solver.py`

Implements the custom INLA solver used for estimation. It optimises the latent field, profiles and integrates over the random-effect precision, constructs Gaussian marginal posterior approximations, and calculates posterior means, variances, covariances and credible intervals.

The solver uses a reduced system based on analytically eliminating the random-effects block through a Schur complement. This avoids constructing and factorising the full latent precision matrix for the very large number of track effects.

The solver loads and saves fitted states as Python pickle files. For the main model, the cache location is set by `GLMM_run.py` to `model-cache/inla-cache.pkl`.

### `gaussian_log1p.py`

Defines the Gaussian likelihood with an identity link for the `log(1 + Streams)` response. In the reported workflow it is used for the preliminary pilot fit, with a fixed observation variance.

The pilot posterior means are used to calculate demeaned residuals, from which the final mixed-Gaussian likelihood parameters are estimated.

### `gaussian_mixed.py`

Defines the mixed-Gaussian likelihood used by the final model. It represents residual variation as a weighted mixture of a narrow Gaussian component and a wider Gaussian component.

When pilot residuals are supplied, the class estimates the component weight and variances iteratively. It provides the likelihood, score, curvature, predictive-distribution and simulation methods required by `INLA_solver.py`.

### `fixed_effects.csv`

Contains the posterior mean, posterior standard deviation and 95% credible interval for the intercept and each fixed-effect coefficient from the main fitted model. These are the estimates reported in Table 4 of the paper.

### `country_slopes_pooled.csv`

Contains the separately fitted track-age slopes for every country and complete year from 2019 to 2025, together with the lower and upper bounds of their 95% credible intervals. These values underlie the country-level model figure discussed in the paper.

### `model-cache/inla-cache.pkl`

Contains the cached posterior state for the main model reported in the paper. The cache includes:

- the posterior intercept mean and variance;
- fixed-effect posterior means, variances and covariance matrix;
- track-level random-effect posterior means and variances;
- candidate random-effect precision values and their integration weights;
- the conditional fixed- and random-effect Gaussian states; and
- likelihood response-parameter states.

The cache supplies the fitted posterior state used for posterior summaries and posterior predictive analysis. It is stored using Git Large File Storage because it exceeds GitHub's standard per-file size limit. It is a Python pickle file and should only be loaded in a trusted environment.

## Python dependencies

The scripts use:

- NumPy
- pandas
- SciPy
- Matplotlib
- tqdm

The full models and predictive checks cannot be rerun without the proprietary Luminate input data. The supplied CSV files contain the numerical results reported in the paper, the combined image presents the prior and posterior predictive checks, and the cache preserves the fitted posterior state from the main model.
